HTML Basics  ï‚Ä¢·¥•‚Ä¢ î

Long story short, HTML scripts are series of multiple enumerated, nested and wrapped elements.
An HTML element is a thing with tags and content, like this: <p>This is a paragraph</p>.
Elements can also have attributes. For example <p class="idk">This is a paragraph</p> has the attribute "class" with the value "idk".
Elements can be nested, but the tags should close properly (they should be "hugging", not "seqential" or "alternative").
Note: Attribute values should generally always be given within double-quotes, even though they're not always necessary for parsing. It's just best practice.
Elements can also be empty, such as "<img src=...>", which doesn't wrap content to affect it - it just embeds an image. This element has no closing tag and no content, and it's called an "empty element".

  • <html> is called the "root element"

  • property value - after a colon. This chooses one of the many preset values that a property can take

More fun facts about these properties:

  • When you set only two values on a property like margin or padding, the first one affects the top and bottom, while the second one is for left and right. The 'auto' keyword devides the available space evenly between two sides.
  • All browsers apply default styling to elements. So that's why even though I'm not defining specific fonts, paddings, etc. for the first part of this page, they must still be styled if you think about it. Also, that's why some pages look different on different browsers - the defaults are not always the same. For example if you open this page on Mozilla and then on Chrome, you'll notice slight differences, due to different defaults (such as the fonts, default padding for headers, etc).
  • The display property - I'm using that for images, I just realised that! So, to begin with, the <body> element is a 'block' - meaning it takes up space on the page. The margin applied to a block element will be respected by other elements inside the page. In contrast, images are by default 'inline' elements.
